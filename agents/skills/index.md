# Ecosy Styled Architecture

This document guides how to develop cross-platform UI components based on the `@ecosy/styled` architecture.

<instructions>
  <rule>
    <title>Platform-Specific Imports</title>
    <details>
      - **React Web**: Absolutely do NOT import pre-wrapped components like `Box`, `Text`, or `View` from `@ecosy/styled/react`. You must generate them using the factory HOC: `const Box = styled("div");`.
      - **React Native**: You MUST import the wrapped primitives directly from `@ecosy/styled/react-native` (e.g., `import { View, Text, Image } from "@ecosy/styled/react-native";`).
    </details>
  </rule>

  <rule>
    <title>Shorthand Paradigm</title>
    <details>
      Always use the provided shorthand properties (`p`, `m`, `px`, `bg`, `c`, `r`, `fs`, `w`, `h`) instead of traditional CSS properties. Never write raw CSS strings or generic StyleSheet objects unless strictly necessary.
    </details>
  </rule>

  <rule>
    <title>Responsive and Platform Selection (sx Prop)</title>
    <details>
      Leverage the `sx` prop for dynamic overrides. The engine supports responsive breakpoints (`base`, `md`, `lg`) and platform selection (`web`, `ios`, `android`) directly inside the sx prop.
      Example: `sx={{ p: { base: 10, md: 20 }, bg: { web: "primary.50", ios: "secondary.50" } }}`.
    </details>
  </rule>

  <rule>
    <title>Dynamic Variants (CVA Alternative)</title>
    <details>
      For components that require multiple states (e.g., Buttons, Badges), ALWAYS use the natively integrated `variants()` configuration provided as the 3rd argument in the `styled()` HOC. Do not use third-party libraries like `classnames` or `clsx`.
    </details>
  </rule>

  <rule>
    <title>Theme Mutation via @ecosy/store</title>
    <details>
      Do not mutate the theme object manually. Always dispatch actions via `@ecosy/store` reducers (e.g., `toggleTheme` or `setTheme`) generated by `getThemeSlice`.
    </details>
  </rule>
</instructions>

<examples>
  <example>
    <description>Standard Web Component with Variants</description>
    <code>
import { styled } from "@ecosy/styled/react";

const Button = styled("button", 
  {
    px: 16, py: 8, r: 8, border: "none", cursor: "pointer",
  },
  {
    variants: {
      intent: { primary: { bg: "primary.50" }, danger: { bg: "red" } },
    },
    defaultVariants: { intent: "primary" }
  }
);
    </code>
  </example>

  <example>
    <description>Standard React Native Component with sx Override</description>
    <code>
import { View, Text, styled } from "@ecosy/styled/react-native";

const Container = styled(View, {
  bg: "background",
  p: 20,
  alignItems: "center",
  flex: 1
});

export const Card = () => (
  <Container sx={{ bg: { ios: "blue", android: "green" } }}>
    <Text sx={{ c: "text", fs: 16 }}>Hello Mobile!</Text>
  </Container>
);
    </code>
  </example>
</examples>
